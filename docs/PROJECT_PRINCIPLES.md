# Продуктовые принципы

## Решать локальную боль

Проект помогает жителям конкретного дома быстро превращать бытовое описание
проблемы в заявку для управляющей организации. Решения оцениваются по пользе в
этом сценарии, а не по широте возможного рынка.

## Сокращать путь до результата

Основной путь состоит из четырёх действий: открыть форму, вставить сообщение,
получить заявку и скопировать её. Интерфейс и архитектура не должны добавлять
шаги без доказанной необходимости.

## Одна проблема — одна заявка

Одна генерация описывает одну наблюдаемую проблему. Несвязанные проблемы не
объединяются в общий текст.

## Не зависеть от канала подачи

Веб-форма работает независимо от Telegram, MAX и других мессенджеров. Готовый
текст можно скопировать в любой доступный канал обращения.

## Начинать с одного дома

Первая версия учитывает потребности одного дома. Универсализация допустима после
появления подтверждённых повторяющихся сценариев.

## Не считать LLM источником истины

Модель преобразует переданные пользователем факты, но не устанавливает причины,
виновников, повреждения или нормы права. Недостающие сведения не выдумываются.

## Отказывать контролируемо

Понятная ошибка безопаснее правдоподобного, но выдуманного результата. При
невалидном вводе или недоступной генерации система возвращает явную ошибку.

## Не универсализировать заранее

Новые пакеты, абстракции и режимы добавляются только для уже возникшей задачи.
Будущие возможности не получают пустые заготовки.

## Оставаться модульным монолитом

Один интерфейс, один серверный процесс и один контейнер остаются базовой
архитектурой до появления измеримой причины разделять систему.
