# ADR-0002: TypeScript и Next.js

## Статус

Accepted

## Контекст

Проекту нужны адаптивная веб-форма и небольшой HTTP API без отдельного backend.
Контракты ввода должны одинаково проверяться в интерфейсе и на сервере.

## Решение

Использовать TypeScript в строгом режиме, Next.js с App Router и Route Handlers,
а Zod — для runtime-валидации общих контрактов.

## Последствия

Интерфейс и API собираются одним приложением. Общие типы уменьшают расхождения
между слоями. Проект принимает соглашения Next.js, но доменный пакет от них не
зависит.
